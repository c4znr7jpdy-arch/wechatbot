# 更新日志

## v2.1.1

增强改进：

- 优化歌词发送流程：当已存储的歌词字段是一个 URL 时，可透明地解析远程歌词内容并进行发送。

## v2.1.0

通过插件配置而非硬编码的方式，使媒体发送模式的平台支持变为可配置。

新功能：

- 允许通过插件设置来配置哪些平台支持录制和文件发送模式。

增强改进：

- 重构发送模式支持检查，从使用硬编码的平台集合改为使用由配置驱动的平台列表。

### v2.0.9

更新音乐插件的配置路径并扩展发送者模式的平台支持，同时为基于 URL 的音频发送增加安全检查。

Bug 修复：

- 修正插件字体和数据目录的解析逻辑，改为使用插件专用路径。
- 当歌曲没有音频 URL 时，避免尝试执行基于 URL 的音频发送。

功能增强：

- 将录音和文件发送模式的支持矩阵扩展到更多消息平台。

### v2.0.8

支持自定义歌曲发送格式，包括卡片、语音、文件、文本等多种格式，满足不同场景下的发送需求。

**使用方式**:

当唤出歌曲列表后,输入<列表序号> <模式> 即可选择指定模式发送歌曲

例如：

`1 卡片`  -> 发送列表第一首歌曲为卡片格式

`2 语音`  -> 发送列表第二首歌曲为语音格式

`3 文件`  -> 发送列表第三首歌曲为文件格式

`4 文本`  -> 发送列表第四首歌曲为文本格式

`1 1` -> 发送列表第一首歌曲为卡片格式

`2 2` -> 发送列表第二首歌曲为语音格式

`3 3` -> 发送列表第三首歌曲为文件格式

`4 4` -> 发送列表第四首歌曲为文本格式

`1 card` -> 发送列表第一首歌曲为卡片格式

`2 record` -> 发送列表第二首歌曲为语音格式

`3 file` -> 发送列表第三首歌曲为文件格式

`4 text` -> 发送列表第四首歌曲为文本格式
