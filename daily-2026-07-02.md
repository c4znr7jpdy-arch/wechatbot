# Daily Report 2026-07-02

## Changes

- Improved live group-chat interjection in `data/plugins/astrbot_plugin_proactive_chat`.
- Added a dedicated live-interjection context window built from platform message history:
  - Messages are sorted by their real message timestamp.
  - Messages older than the configured maximum lookback are ignored.
  - While walking backward from the trigger message, any gap longer than `topic_gap_minutes` cuts off older context.
  - The default topic gap is `5` minutes.
- Updated the live-interjection prompt to inject `{{live_context}}` / `{{context_window}}` and explicitly avoid reviving old discarded topics.
- Added backward compatibility for already-saved old interjection prompts: if the prompt lacks a live-context placeholder, the current continuous chat window is prepended automatically.
- Added image-aware context formatting for interjection:
  - Reuses existing image captions already present in message text.
  - Attempts VLM captioning through AstrBot `context.llm_generate(..., image_urls=[...])` when an image reference is available.
  - Uses `openai_2/mimo-v2.5` by default, matching the project image-caption provider.
  - Adds a small in-memory caption cache to avoid repeatedly describing the same image.
- Changed the trigger freshness check to use the original message timestamp instead of the event processing time.
- Added WebUI config schema fields for interjection context size, lookback, topic gap, max chars, image caption limit, image caption timeout, provider id, and caption prompt.
